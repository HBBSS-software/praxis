# Social Practice Record System

## Stack

- Bun for dependency management, scripts, and the backend runtime
- TypeScript for backend code, frontend page logic, and tests
- Express for the API
- A JSON file store in `backend/database.json`
- Static HTML and CSS for the frontend

## Development

Install dependencies:

```bash
bun install
```

Start the backend:

```bash
bun run dev
```

Build the frontend browser modules:

```bash
bun run build:frontend
```

Run type checks and tests:

```bash
bun run test
```

The backend listens on `http://localhost:3000`.

Open `frontend/login.html` in a browser after the frontend build has been generated.

## Security Settings

Set these environment variables before deployment:

```bash
JWT_SECRET=replace-with-a-long-random-secret-at-least-32-characters
CORS_ORIGINS=http://localhost:3000,http://127.0.0.1:3000
JWT_EXPIRES_IN=8h
LOGIN_MAX_ATTEMPTS=5
LOGIN_LOCKOUT_MS=900000
```

The frontend stores auth state in `sessionStorage` instead of `localStorage`, and the backend now adds stricter JWT validation, login rate limiting, cache-control headers for auth responses, and `helmet` security headers.
